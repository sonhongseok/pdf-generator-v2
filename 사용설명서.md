# OP 성적서 발행기 — 사용 설명서

> 주식회사 옵티로(Optilo Co., Ltd.) 교정 성적서(Calibration Certificate) 자동 발급 시스템 사용 안내서입니다.

---

## 1. 이 프로그램이 하는 일

교정 성적서 PDF를 자동으로 만들어주는 사내 전용 웹 시스템입니다.

- 발행일, 교정일, 만료일, 시리얼 번호를 입력하면
- MS Word 성적서 양식에 자동으로 데이터를 채워
- **PDF 파일을 즉시 다운로드**합니다.
- 발급 이력이 데이터베이스에 자동 저장됩니다.

---

## 2. 프로그램 접속 방법

별도의 설치나 실행 없이, **웹 브라우저(Chrome 권장)에서 아래 주소를 입력**하면 바로 사용할 수 있습니다.

```
http://[서버 IP 주소]
```

> [!IMPORTANT]
> 접속 주소는 시스템 담당자에게 문의하세요. 사내 네트워크(Wi-Fi 또는 유선)에 연결된 상태에서만 접속 가능합니다.

---

## 3. 화면 입력 항목 설명

접속하면 아래와 같은 입력 화면이 표시됩니다.

```
+-----------------------------------------------+
|  Certificate Date  [ 2026-05-26 ]             |
|  Calibration Date  [ 2026-05-20 ]             |
|  Expiry Date       [           ]              |
|  Serial No         [           ]              |
|                                               |
|  [ 성적서 생성 ]          [ 종료 ]             |
+-----------------------------------------------+
```

### 3-1. Certificate Date (발행일)
- 오늘 이 성적서를 발행하는 날짜입니다.
- 페이지 접속 시 오늘 날짜로 자동 입력됩니다.
- 입력 형식: `2026-05-26` 또는 `20260526` (8자리 숫자 입력 시 자동 변환)

### 3-2. Calibration Date (교정일)
- 실제 교정 작업이 이루어진 날짜입니다.
- 페이지 접속 시 오늘 날짜로 자동 입력됩니다.
- 교정이 며칠 전에 수행된 경우, 해당 날짜로 변경하여 입력하세요.

### 3-3. Expiry Date (만료일)
- 성적서 유효 기간이 끝나는 날짜입니다.
- 반드시 직접 입력해야 합니다. (자동 입력 없음)
- 만료일은 발행일과 교정일보다 반드시 미래 날짜여야 합니다.

### 3-4. Serial No (시리얼 번호)
- 교정 대상 장비의 고유 시리얼 번호입니다.
- 한 개만 발급: `SN-12345`
- 여러 개 동시 발급: **스페이스(공백)로 구분** (예: `SN-001 SN-002 SN-003`)
- 여러 개 입력 시 시리얼 순서대로 페이지가 생성되어 **하나의 PDF 파일**로 합쳐집니다.

---

## 4. 성적서 생성 절차

### STEP 1 - 정보 입력
Certificate Date, Calibration Date, Expiry Date, Serial No 4가지를 모두 입력합니다.

### STEP 2 - 생성 버튼 클릭
화면 하단 [성적서 생성] 버튼을 클릭합니다.

### STEP 3 - 진행 상태 확인

| 상태 메시지 | 의미 |
|---|---|
| `Ready` | 대기 중. 입력 후 버튼을 클릭하세요. |
| `Generating PDF Certificates...` | PDF 변환 작업 중입니다. 잠시 기다려 주세요. |
| `Success: PDF generated successfully.` | [성공] PDF 파일이 자동으로 다운로드됩니다. |
| `Error: ...` | [오류] 화면 하단 Warning 메시지를 확인하세요. |

### STEP 4 - PDF 다운로드 확인
성공 시 PDF 파일이 브라우저 기본 다운로드 폴더에 자동 저장됩니다.
파일명 형식: `OP발행일0001.pdf` (예: `OP202605260001.pdf`)

---

## 5. Certificate NO. 번호 구성 방식

사용자가 번호를 직접 입력하지 않아도 자동 생성됩니다.

```
OP  +  발행일(8자리)  +  페이지 순번(4자리)
예:    OP   20260526   0001
결과:  OP202605260001
```

시리얼 번호를 3개 입력한 경우:

| 페이지 | 시리얼 번호 | Certificate NO. |
|---|---|---|
| 1페이지 | SN-001 | OP202605260001 |
| 2페이지 | SN-002 | OP202605260002 |
| 3페이지 | SN-003 | OP202605260003 |

---

## 6. 오류 메시지 해결 방법

화면 하단에 Warning 경고 박스가 나타나면 아래 표를 참고하세요.

| 오류 메시지 | 원인 | 해결 방법 |
|---|---|---|
| `Certificate Date를 입력해 주세요.` | 발행일 미입력 | Certificate Date 칸에 날짜 입력 |
| `Expiry Date를 입력해 주세요.` | 만료일 미입력 | Expiry Date 칸에 날짜 입력 |
| `Serial No를 입력해 주세요.` | 시리얼 번호 미입력 | Serial No 칸에 시리얼 번호 입력 |
| `날짜 형식이 올바르지 않습니다.` | 날짜 형식 오류 | `2026-05-26` 또는 `20260526` 형식으로 재입력 |
| `만료일은 발행일/교정일보다 빠를 수 없습니다.` | 만료일이 과거 날짜 | 만료일을 더 미래의 날짜로 수정 |
| `이미 발급된 성적서가 존재합니다.` | 완전 중복 발급 시도 | 입력 정보 재확인. 동일 내역은 재발급 불가 |
| `서버 내부 에러로 PDF 생성을 진행할 수 없습니다.` | 서버 오류 | 시스템 담당자에게 문의 |

---

## 7. 성적서 양식 수정 방법

> [!IMPORTANT]
> 성적서 양식(디자인, 로고, 문구 등)을 변경할 때는 시스템 담당자에게 요청하세요.
> 양식 수정은 서버에 있는 템플릿 파일(`certificate_template.docx`)을 수정하면 됩니다. 수정 후 서버 재시작 없이 즉시 반영됩니다.

### 담당자가 수정할 경우 절차

1. 서버 접근 후 아래 경로의 파일을 MS Word로 엽니다.

```
[서버 설치 경로]/certificate_template.docx
```

2. 로고, 폰트, 문구, 표 레이아웃 등을 자유롭게 수정합니다.
3. 아래 변수 태그는 반드시 그대로 유지해야 합니다.

| 태그 | 치환되는 내용 |
|---|---|
| `{cno}` | Certificate NO. |
| `{sno}` | Serial No. (장비 시리얼 번호) |
| `{cedate}` | Certificate Date (발행일) |
| `{pdate}` | Calibration Date (교정일) |
| `{edate}` | Expiry Date (만료일) |

4. 저장 후 바로 다음 성적서 생성 시 적용됩니다. (서버 재시작 불필요)

## 8. 자주 묻는 질문 (FAQ)

**Q. 시리얼 번호 여러 개를 입력할 때 구분 기호가 뭔가요?**
- 스페이스(공백)로 구분합니다. 쉼표(,)는 사용하지 마세요.
- 올바른 예: `SN-001 SN-002 SN-003`

**Q. PDF가 다운로드됐는데 어디에 저장되나요?**
- 브라우저의 기본 다운로드 폴더에 저장됩니다. (보통 `내 PC > 다운로드`)
- Chrome 우측 상단 다운로드 아이콘을 클릭하면 확인할 수 있습니다.

**Q. 날짜를 `20260526`처럼 붙여서 입력해도 되나요?**
- 네, 됩니다. 8자리 숫자 입력 시 `2026-05-26` 형식으로 자동 변환됩니다.

**Q. 같은 날짜에 같은 시리얼로 또 발급하면 어떻게 되나요?**
- 완전히 동일한 조합은 중복 발급 방지 기능에 의해 차단됩니다.
- 화면에 `이미 발급된 성적서가 존재합니다.` 경고가 표시됩니다.

**Q. 접속이 안 되거나 페이지가 열리지 않아요.**
- 사내 네트워크(Wi-Fi 또는 유선)에 연결되어 있는지 확인하세요.
- 네트워크 연결 이후에도 접속이 안 되면 시스템 담당자에게 문의하세요.

**Q. 여러 사람이 동시에 사용해도 되나요?**
- 네, 동시에 여러 명이 사용할 수 있습니다.
- 같은 사내 네트워크라면 PC, 노트북 모두 동일한 URL로 접속 가능합니다.
