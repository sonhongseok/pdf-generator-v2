# 📋 OP 성적서 발행기 — 사용 설명서

> 이 설명서는 **프로그램을 처음 사용하거나, 날마다 업무에 활용하는 담당자**를 위한 안내서입니다.
> 개발 지식이 없어도 이 문서만 보고 100% 활용할 수 있도록 작성되었습니다.

---

## 1. 이 프로그램이 하는 일

교정 성적서(Calibration Certificate) PDF를 자동으로 만들어주는 사내 전용 도구입니다.

- 발행일, 교정일, 만료일, 시리얼 번호를 입력하면
- MS Word 성적서 양식에 자동으로 데이터를 채워
- **PDF 파일을 즉시 다운로드**해 드립니다.
- 동시에 발급 이력이 데이터베이스에 자동으로 기록됩니다.

---

## 2. 프로그램 시작 방법

### 방법 1 — 배치파일 실행 (가장 쉬움)

`C:\work\pdf-generator` 폴더 안에 있는 아래 파일을 **더블클릭**합니다.

```
📁 C:\work\pdf-generator\
   └── 📄 OP_성적서발행기.bat  ← 이 파일을 더블클릭
```

배치파일이 자동으로 아래 단계를 순서대로 수행합니다.
1. 백엔드 서버(Spring Boot)가 이미 켜져 있는지 확인 → 꺼져 있으면 자동 시작
2. 프론트엔드 서버(Vite)가 이미 켜져 있는지 확인 → 꺼져 있으면 자동 시작
3. Chrome 브라우저를 **앱 모드(브라우저 주소창 없음)**로 자동으로 열어줌

> [!NOTE]
> 배치파일을 실행했을 때 검은 콘솔 창이 잠깐 보이다가 Chrome 창이 열리면 정상입니다.
> 첫 실행 시 백엔드가 완전히 켜지는 데 최대 15~20초 정도 걸릴 수 있습니다.

---

### 방법 2 — 수동으로 각각 실행

배치파일이 실행되지 않는 경우, 아래 순서대로 수동으로 실행합니다.

#### ① 백엔드 시작

```
PowerShell 또는 cmd를 열고 아래 명령어를 실행
cd C:\work\pdf-generator\backend
.\mvnw.cmd spring-boot:run
```

`Started PdfGeneratorApplication in ...` 메시지가 보이면 성공입니다.

#### ② 프론트엔드 시작

새로운 PowerShell 또는 cmd 창을 열고 실행합니다.

```
cd C:\work\pdf-generator\frontend
npm run dev
```

`http://localhost:5173/` 가 표시되면 성공입니다.

#### ③ 브라우저에서 접속

Chrome 브라우저에서 `http://localhost:5173` 주소를 입력하고 접속합니다.

## 3. 화면 입력 항목 상세 설명

프로그램 화면에는 아래 4가지 항목을 입력하는 칸이 있습니다.

```
┌─────────────────────────────────────┐
│  Certificate Date  [ 2026-05-26 ]   │
│  Calibration Date  [ 2026-05-20 ]   │
│  Expiry Date       [           ]   │
│  Serial No         [           ]   │
│                                     │
│  [ 성적서 생성 ]      [ 종료 ]       │
└─────────────────────────────────────┘
```

---

### 3-1. Certificate Date (발행일)

- **뜻**: 오늘 이 성적서를 발행하는 날짜입니다.
- **프로그램 시작 시 오늘 날짜로 자동 입력**됩니다.
- **입력 형식**: `YYYY-MM-DD` (예: `2026-05-26`)
  - 또는 붙여쓰기로 `20260526`을 입력해도 자동으로 변환됩니다.

---

### 3-2. Calibration Date (교정일)

- **뜻**: 실제 교정 작업이 이루어진 날짜입니다.
- **프로그램 시작 시 오늘 날짜로 자동 입력**됩니다.
- 교정이 며칠 전에 수행된 경우, 그 날짜로 변경해서 입력합니다.
- **입력 형식**: `YYYY-MM-DD` (예: `2026-05-20`)

---

### 3-3. Expiry Date (만료일)

- **뜻**: 이 성적서의 유효 기간이 끝나는 날짜입니다.
- **반드시 직접 입력**해야 합니다 (자동 입력 없음).
- 만료일은 발행일 / 교정일보다 **반드시 미래 날짜**여야 합니다. 과거 날짜를 입력하면 오류가 납니다.
- **입력 형식**: `YYYY-MM-DD` (예: `2027-05-20`)

---

### 3-4. Serial No (시리얼 번호)

- **뜻**: 교정 대상 장비의 고유 시리얼 번호입니다.
- **한 개만 발급할 경우**: 시리얼 번호 1개만 입력합니다. (예: `SN-12345`)
- **여러 개를 한 번에 발급할 경우**: **스페이스(공백)로 구분**하여 여러 개를 입력합니다.
  - 예: `SN-001 SN-002 SN-003`
  - 입력한 순서대로 PDF 내 페이지가 생성되어 **하나의 PDF 파일로 합쳐** 저장됩니다.

---

## 4. 성적서 생성 절차 (단계별)

### STEP 1 — 정보 입력

각 항목을 정확하게 입력합니다.

> [!IMPORTANT]
> Certificate Date, Calibration Date, Expiry Date, Serial No 4가지는 모두 필수 입력 항목입니다.

### STEP 2 — 생성 버튼 클릭

화면 하단의 **[성적서 생성]** 버튼을 클릭합니다.

### STEP 3 — 진행 상태 확인

버튼 위의 상태 표시줄에서 진행 현황을 확인합니다.

| 상태 메시지 | 의미 |
|---|---|
| `Ready` | 대기 중. 입력 후 버튼을 클릭하세요. |
| `Generating PDF Certificates...` | PDF 변환 작업 중. 잠시 기다리세요. |
| `Success: PDF generated successfully.` | ✅ 성공! 파일이 다운로드 됩니다. |
| `Error: ...` | ❌ 오류 발생. 아래 Warning 메시지를 확인하세요. |

### STEP 4 — PDF 다운로드 확인

성공하면 PDF 파일이 브라우저 기본 다운로드 폴더(보통 `C:\Users\사용자명\Downloads`)에 자동으로 저장됩니다.

- 파일명은 자동으로 **`OP발행일0001.pdf`** 형태로 지정됩니다.
- 예시: `OP202605260001.pdf`

## 5. Certificate NO. 번호 구성 방식

사용자가 번호를 직접 입력하지 않아도 자동으로 생성됩니다.

```
OP  +  발행일(8자리)  +  페이지 순번(4자리)
 ↓         ↓                  ↓
OP   20260526         0001
            ▶ 최종 결과: OP202605260001
```

시리얼 번호를 3개 입력한 경우, PDF 파일 안에는 아래처럼 3개의 페이지가 포함됩니다.

| 페이지 | 시리얼 번호 | Certificate NO. |
|---|---|---|
| 1페이지 | SN-001 | OP202605260001 |
| 2페이지 | SN-002 | OP202605260002 |
| 3페이지 | SN-003 | OP202605260003 |

## 6. 오류 메시지 해결 방법

### ⚠️ Warning 메시지가 화면에 표시되는 경우

화면 하단에 주황색(Warning) 경고 박스가 나타나면 아래 표를 참고하여 해결하세요.

| 오류 메시지 | 원인 | 해결 방법 |
|---|---|---|
| `Certificate Date를 입력해 주세요.` | 발행일이 비어 있음 | Certificate Date 칸에 날짜를 입력하세요 |
| `Calibration Date를 입력해 주세요.` | 교정일이 비어 있음 | Calibration Date 칸에 날짜를 입력하세요 |
| `Expiry Date를 입력해 주세요.` | 만료일이 비어 있음 | Expiry Date 칸에 날짜를 입력하세요 |
| `Serial No를 입력해 주세요.` | 시리얼 번호가 비어 있음 | Serial No 칸에 시리얼을 입력하세요 |
| `날짜 형식이 올바르지 않습니다.` | 날짜 형식이 잘못됨 | `2026-05-26` 또는 `20260526` 형식으로 재입력하세요 |
| `만료일은 발행일/교정일보다 빠를 수 없습니다.` | 만료일이 발행일보다 과거 | 만료일을 더 미래의 날짜로 수정하세요 |
| `동일한 발행일, 교정일, 만료일, 시리얼 번호로 이미 발급된 성적서가 존재합니다.` | 중복 발급 시도 | 이미 같은 내용으로 발급된 이력이 있습니다. 입력 정보를 다시 확인하세요 |
| `서버 내부 에러로 PDF 생성을 진행할 수 없습니다.` | 백엔드 서버 오류 | 아래 [서버가 응답하지 않는 경우] 항목을 참고하세요 |

---

### 🔴 서버가 응답하지 않는 경우 (프로그램이 아예 안 열리거나, 생성 버튼을 눌러도 아무 반응이 없는 경우)

> [!WARNING]
> 아래 순서대로 확인 후 해결되지 않으면 개발자에게 문의하세요.

#### 확인 1 — MySQL 데이터베이스가 실행 중인지 확인

1. Windows 키를 눌러 **`서비스`** 를 검색하여 실행합니다.
2. 목록에서 `MySQL80` (또는 `MySQL`)을 찾습니다.
3. 상태가 **`실행 중`** 인지 확인합니다. 멈춰 있으면 우클릭 → **`시작`** 을 클릭합니다.

#### 확인 2 — 배치파일을 다시 실행

배치파일(`OP_성적서발행기.bat`)을 다시 더블클릭합니다.
이미 실행 중인 경우에는 아무것도 중복으로 켜지지 않습니다.

#### 확인 3 — 백엔드가 정상적으로 켜졌는지 직접 확인

Chrome 브라우저에서 `http://localhost:8080` 주소를 입력해봅니다.
빈 페이지나 어떤 내용이라도 보이면 백엔드가 켜져 있는 것입니다.

## 7. 성적서 양식 수정 방법

> [!IMPORTANT]
> 성적서의 디자인, 로고, 문구, 폰트 등을 변경하고 싶을 때는 **코드 수정 없이** Word 파일만 바꾸면 됩니다.

### 수정 대상 파일

```
C:\work\pdf-generator\certificate_template.docx
```

### 수정 절차

1. `C:\work\pdf-generator\` 폴더에서 **`certificate_template.docx`** 파일을 더블클릭하여 MS Word로 엽니다.
2. 원하는 내용을 자유롭게 수정합니다.
   - 폰트 변경, 크기 조정, 문구 수정, 로고 이미지 교체 등 모두 가능합니다.
3. **반드시 아래 태그(변수)는 그대로 유지**해야 합니다. 이 태그를 지우면 오류가 발생합니다.

| 태그 | 치환되는 내용 |
|---|---|
| `{cno}` | Certificate NO. (예: OP202605260001) |
| `{sno}` | Serial No. (장비 시리얼 번호) |
| `{cedate}` | Certificate Date (발행일) |
| `{pdate}` | Calibration Date (교정일) |
| `{edate}` | Expiry Date (만료일) |

4. 수정이 완료되면 **파일 저장** 후 닫습니다.
5. **서버를 재시작하지 않아도** 다음 번 성적서 생성 시 즉시 새로운 양식이 적용됩니다.

> [!CAUTION]
> `{cno}`, `{sno}` 등의 태그를 실수로 삭제하면 PDF 생성 시 빈 칸으로 출력되거나 오류가 날 수 있습니다.
> 수정 전에 파일을 복사본으로 백업해두는 것을 권장합니다.

화면 우측 하단의 **[종료]** 버튼을 클릭합니다.

> 서버까지 완전히 종료하려면 배치파일 실행 시 열렸던 콘솔 창(검은 창)을 직접 닫으세요.

## 9. 자주 묻는 질문 (FAQ)

**Q. 시리얼 번호 여러 개를 입력할 때 구분 기호가 뭔가요?**
- **스페이스(공백)**로 구분합니다. 쉼표(,)는 사용하지 마세요.
- 올바른 예: `SN-001 SN-002 SN-003`
- 잘못된 예: `SN-001, SN-002, SN-003`

---

**Q. PDF가 다운로드됐는데 어디에 저장되나요?**
- Chrome 브라우저의 기본 다운로드 폴더에 저장됩니다.
- 일반적으로 `C:\Users\사용자이름\Downloads\` 폴더입니다.
- Chrome 오른쪽 상단 ⬇ 아이콘을 클릭하면 최근 다운로드된 파일을 확인할 수 있습니다.

---

**Q. 날짜를 `20260526` 처럼 붙여서 입력해도 되나요?**
- 네, 됩니다! 8자리 숫자를 입력하면 자동으로 `2026-05-26` 형식으로 변환해 드립니다.

---

**Q. 성적서 양식의 로고 이미지를 바꾸고 싶어요.**
- `certificate_template.docx` 파일을 MS Word에서 열고, 로고 이미지를 클릭하여 선택한 다음 새 이미지로 교체하면 됩니다. (코드 수정 불필요)

---

**Q. 같은 날짜에 같은 시리얼로 또 발급을 하면 어떻게 되나요?**
- 완전히 동일한 발행일, 교정일, 만료일, 시리얼 번호 세트는 **중복 발급 방지 로직**에 의해 자동으로 차단됩니다.
- 화면에 `이미 발급된 성적서가 존재합니다.` 경고 메시지가 표시됩니다.

---

**Q. 프로그램을 여러 사람이 동시에 사용해도 되나요?**
- 네, 가능합니다. 단, 서버가 같은 PC에서 실행 중이라면 해당 PC의 주소로 접속해야 합니다.
  - 다른 PC에서는 `http://[서버PC의IP주소]:5173` 주소로 접속합니다.
