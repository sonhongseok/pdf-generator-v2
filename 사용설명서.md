# OP 성적서 발행기 — 사용 설명서

> 주식회사 옵티로(Optilo Co., Ltd.) 교정 성적서(Calibration Certificate) 자동 발급 시스템 사용 안내서입니다.

---

## 1. 이 프로그램이 하는 일

교정 성적서 PDF를 자동으로 만들어주는 사내 전용 웹 시스템입니다.

- 발행일, 교정일, 만료일, 시리얼 번호를 입력하면
- MS Word 성적서 양식에 자동으로 데이터를 채워
- **PDF 파일을 즉시 다운로드**합니다.
- 발급 이력이 데이터베이스에 자동 저장되며, 언제든 다시 다운로드할 수 있습니다.

---

## 2. 프로그램 접속 방법

`OP_성적서발행기.bat` 파일을 더블클릭하면 서버가 자동 실행되고 프로그램 화면이 열립니다.

> [!IMPORTANT]
> 프로그램 실행 후 화면이 열리기까지 최초 30~60초 정도 소요될 수 있습니다.
> Microsoft Word와 MySQL 서버가 PC에 설치되어 있어야 정상 작동합니다.

---

## 3. 화면 구성

프로그램 화면 상단에는 두 개의 탭이 있습니다.

```
[ 성적서 발급 ]   [ 발급 이력 조회 ]
```

- **성적서 발급 탭**: 새 성적서를 생성하고 다운로드합니다.
- **발급 이력 조회 탭**: 과거에 발급한 성적서 목록을 확인하고 재다운로드합니다.

---

## 4. 성적서 발급 탭 — 입력 항목 설명

```
+---------------------------------------------------+
|  Certificate Date  [ 2026-06-05 ]                 |
|  Calibration Date  [ 2026-06-05 ]                 |
|  Expiry Date       [ 2027-06-05 ]                 |
|  Serial No         [                          ]   |
|  생성 방식          ● 통합 PDF  ○ 개별 PDF (ZIP)   |
|  Start No          [      ] (0001~9999, 선택)      |
|                                                   |
|  [ 성적서 생성 ]            [ 종료 ]               |
+---------------------------------------------------+
```

### 4-1. Certificate Date (발행일)
- 이 성적서를 발행하는 날짜입니다.
- 프로그램 실행 시 **오늘 날짜가 자동 입력**됩니다.
- 입력 형식: `2026-06-05` 또는 `20260605` (8자리 숫자 입력 시 자동 변환)

### 4-2. Calibration Date (교정일)
- 실제 교정 작업이 이루어진 날짜입니다.
- 프로그램 실행 시 **오늘 날짜가 자동 입력**됩니다.
- 교정이 며칠 전에 수행된 경우, 해당 날짜로 변경하여 입력하세요.
- 교정일을 변경하면 **만료일이 교정일 기준 1년 후로 자동 업데이트**됩니다.

### 4-3. Expiry Date (만료일)
- 성적서 유효 기간이 끝나는 날짜입니다.
- Calibration Date 입력 시 **1년 후 날짜로 자동 입력**됩니다.
- 필요 시 직접 수정할 수 있습니다.
- 만료일은 발행일보다 반드시 미래 날짜여야 합니다.

### 4-4. Serial No (시리얼 번호)
- 교정 대상 장비의 고유 시리얼 번호입니다.
- **한 개 발급**: `SN-12345`
- **여러 개 동시 발급**: 공백(스페이스)으로 구분하여 입력합니다.
  - 예: `SN-001 SN-002 SN-003`
- 쉼표(,)는 사용하지 마세요.

### 4-5. 생성 방식
- **통합 PDF**: 시리얼 개수만큼 페이지를 만들어 **하나의 PDF 파일**로 합쳐서 다운로드합니다. (기본값)
- **개별 PDF (ZIP)**: 시리얼 번호마다 PDF 파일을 **각각 1장씩 생성**하여 하나의 ZIP 압축 파일로 다운로드합니다. ZIP 내부에 `{시리얼번호}.pdf` 형태로 저장됩니다.

### 4-6. Start No (시작 번호, 선택 입력)
- Certificate NO.의 끝 4자리 번호를 직접 지정할 때 사용합니다.
- **비워두면** `0001`부터 자동으로 시작합니다.
- **단일값 입력**: 해당 번호부터 순차 증가합니다. (예: `0050` 입력 → `0050, 0051, 0052...`)
- **다중값 입력**: 공백으로 구분하며, 시리얼 번호 개수와 정확히 일치해야 합니다. (예: `0050 0055 0060`)
- 입력 가능 범위: `0001` ~ `9999`

## 5. 성적서 생성 절차

### STEP 1 — 정보 입력
Certificate Date, Calibration Date, Expiry Date, Serial No를 모두 입력합니다.

### STEP 2 — 생성 방식 선택
통합 PDF 또는 개별 PDF (ZIP) 중 원하는 방식을 선택합니다.

### STEP 3 — 생성 버튼 클릭
화면 하단 **[성적서 생성]** 버튼을 클릭합니다.

### STEP 4 — 최종 확인 팝업
발급 내용 요약 팝업이 뜹니다. 내용을 확인한 후 **[확인]** 을 누르면 발급 이력이 DB에 저장되고 파일 생성이 시작됩니다.

> [!NOTE]
> [확인]을 누르는 순간 발급 이력이 저장됩니다. 취소하려면 [취소]를 누르세요.

### STEP 5 — 진행 상태 확인

| 상태 표시 | 의미 |
|---|---|
| `Ready` | 대기 중. 입력 후 버튼을 클릭하세요. |
| (진행 바 애니메이션) | PDF 변환 작업 중입니다. 잠시 기다려 주세요. |
| `Success: [발급번호] 발급 완료.` | 파일이 자동으로 다운로드됩니다. |
| `Error: ...` | 오류 발생. 화면 하단 Warning 메시지를 확인하세요. |

> [!NOTE]
> MS Word를 통한 PDF 변환 작업은 시리얼 1개당 1~2분이 소요될 수 있습니다.
> 여러 개의 시리얼을 입력한 경우 그만큼 시간이 더 걸립니다.

### STEP 6 — 파일 다운로드 확인
성공 시 파일이 브라우저 기본 다운로드 폴더에 자동 저장됩니다.

| 생성 방식 | 파일명 예시 |
|---|---|
| 통합 PDF | `OP202606050001.pdf` |
| 개별 PDF (ZIP) | `OP202606050001.zip` (내부에 `SN-001.pdf`, `SN-002.pdf` 등) |

## 6. Certificate NO. 번호 구성 방식

사용자가 번호를 직접 입력하지 않아도 자동 생성됩니다.

```
OP  +  발행일(8자리)  +  페이지 순번(4자리)
예:    OP   20260605   0001
결과:  OP202606050001
```

시리얼 번호 3개를 통합 PDF로 발급한 경우 (Start No 입력 없음):

| 페이지 | 시리얼 번호 | Certificate NO. |
|---|---|---|
| 1페이지 | SN-001 | OP202606050001 |
| 2페이지 | SN-002 | OP202606050002 |
| 3페이지 | SN-003 | OP202606050003 |

Start No를 `0050`으로 지정한 경우:

| 페이지 | 시리얼 번호 | Certificate NO. |
|---|---|---|
| 1페이지 | SN-001 | OP202606050050 |
| 2페이지 | SN-002 | OP202606050051 |
| 3페이지 | SN-003 | OP202606050052 |

## 7. 발급 이력 조회 탭

상단의 **[발급 이력 조회]** 탭을 클릭하면 과거 발급 내역 전체를 확인할 수 있습니다.

### 주요 기능

**검색**
- 상단 검색창에 발급 번호 또는 시리얼 번호 일부를 입력하면 실시간으로 필터링됩니다.

**시리얼 번호 펼치기**
- 목록의 각 행을 클릭하면 해당 발급 건에 포함된 시리얼 번호 전체 목록이 펼쳐집니다.

**재다운로드**
- 각 행 우측의 **[통합]** 또는 **[개별(ZIP)]** 버튼을 클릭하면 해당 발급 건을 다시 생성하여 다운로드할 수 있습니다.
- 최초 발급 당시와 동일한 Certificate NO., 날짜, 시리얼 번호로 재생성됩니다.

> [!NOTE]
> 재다운로드도 PDF 변환 과정이 포함되므로 1~2분 소요될 수 있습니다.

## 8. 오류 메시지 해결 방법

화면 하단에 **Warning** 경고 박스가 나타나면 아래 표를 참고하세요.

| 오류 메시지 | 원인 | 해결 방법 |
|---|---|---|
| `Certificate Date를 입력해 주세요.` | 발행일 미입력 | Certificate Date 칸에 날짜 입력 |
| `Expiry Date를 입력해 주세요.` | 만료일 미입력 | Expiry Date 칸에 날짜 입력 |
| `Serial No를 입력해 주세요.` | 시리얼 번호 미입력 | Serial No 칸에 시리얼 번호 입력 |
| `날짜 형식이 올바르지 않습니다.` | 날짜 형식 오류 | `2026-06-05` 또는 `20260605` 형식으로 재입력 |
| `만료일은 발행일/교정일보다 빠를 수 없습니다.` | 만료일이 과거 날짜 | 만료일을 더 미래의 날짜로 수정 |
| `입력한 Serial No 중에 중복된 값이 있습니다.` | 시리얼 번호 중복 | 동일한 시리얼 번호 제거 후 재입력 |
| `Start No는 0001~9999 범위의 4자리 숫자여야 합니다.` | Start No 형식 오류 | 올바른 4자리 숫자 입력 (예: `0001`) |
| `Start No 개수와 Serial No 개수가 일치하지 않습니다.` | Start No 다중 입력 개수 불일치 | Start No를 비우거나 시리얼 개수와 동일하게 입력 |
| `이미 발급된 성적서가 존재합니다.` | 완전 중복 발급 시도 | 입력 정보 재확인. 동일 내역은 재발급 불가 |
| `서버 내부 에러로 PDF 생성을 진행할 수 없습니다.` | 서버 오류 | 시스템 담당자에게 문의 |

## 9. 성적서 양식 수정 방법

> [!IMPORTANT]
> 성적서 양식(디자인, 로고, 문구 등)을 변경할 때는 시스템 담당자에게 요청하세요.
> 양식 수정은 서버에 있는 템플릿 파일(`certificate_template.docx`)을 수정하면 됩니다. 수정 후 서버 재시작 없이 즉시 반영됩니다.

### 담당자가 수정할 경우 절차

1. 프로젝트 루트 경로의 `certificate_template.docx` 파일을 MS Word로 엽니다.
2. 로고, 폰트, 문구, 표 레이아웃 등을 자유롭게 수정합니다.
3. 아래 변수 태그는 반드시 그대로 유지해야 합니다.

| 태그 | 치환되는 내용 |
|---|---|
| `{cno}` | Certificate NO. |
| `{sno}` | Serial No. (장비 시리얼 번호) |
| `{cedate}` | Certificate Date (발행일) |
| `{pdate}` | Calibration Date (교정일) |
| `{edate}` | Expiry Date (만료일) |

4. 저장 후 바로 다음 성적서 생성 시 적용됩니다. (서버 재시작 불필요)

## 10. 자주 묻는 질문 (FAQ)

**Q. 시리얼 번호 여러 개를 입력할 때 구분 기호가 뭔가요?**
- 스페이스(공백)로 구분합니다. 쉼표(,)는 사용하지 마세요.
- 올바른 예: `SN-001 SN-002 SN-003`

**Q. 통합 PDF와 개별 PDF(ZIP)는 어떻게 다른가요?**
- **통합 PDF**: 시리얼 번호 수만큼의 페이지가 하나의 PDF 파일로 합쳐집니다. 한 번에 인쇄하거나 전달할 때 편리합니다.
- **개별 PDF(ZIP)**: 시리얼 번호별로 파일이 각각 분리되어 ZIP으로 압축됩니다. 각 장비별로 따로 보관하거나 전달할 때 편리합니다.

**Q. PDF가 다운로드됐는데 어디에 저장되나요?**
- 브라우저의 기본 다운로드 폴더에 저장됩니다. (보통 `내 PC > 다운로드`)

**Q. 날짜를 `20260605`처럼 붙여서 입력해도 되나요?**
- 네, 됩니다. 8자리 숫자 입력 시 `2026-06-05` 형식으로 자동 변환됩니다.

**Q. 같은 날짜에 같은 시리얼로 또 발급하면 어떻게 되나요?**
- 발행일, 교정일, 만료일, 시리얼 번호가 모두 완전히 동일한 경우 중복 발급이 차단됩니다.
- 화면에 `이미 발급된 성적서가 존재합니다.` 경고가 표시됩니다.

**Q. 과거에 발급한 성적서를 다시 받을 수 있나요?**
- 네, 가능합니다. 상단 **[발급 이력 조회]** 탭에서 해당 이력을 찾아 **[통합]** 또는 **[개별(ZIP)]** 버튼을 누르면 됩니다.

**Q. Start No는 언제 사용하나요?**
- 같은 날 이미 발급한 성적서가 있어서 번호가 겹치지 않도록 시작 번호를 직접 지정해야 할 때 사용합니다.
- 예: 오전에 이미 `0001~0005`를 발급했다면, 오후 발급 시 Start No에 `0006`을 입력합니다.

**Q. 접속이 안 되거나 페이지가 열리지 않아요.**
- `OP_성적서발행기.bat`을 다시 더블클릭해 보세요.
- 그래도 안 되면 시스템 담당자에게 문의하세요.

**Q. 여러 사람이 동시에 사용해도 되나요?**
- 네, 동시에 여러 명이 사용할 수 있습니다.
- 같은 사내 네트워크라면 PC, 노트북 모두 동일한 URL로 접속 가능합니다.
